What does it do?

Makeshift technic, as the name suggests, is a DIY technicpack launcher. For users: install, update, uninstall and manage your favourite modpack from within this launcher. For modders: update your modpack, official server list. Manage mods and configs.

How is it done?


On every launch/ longer periods of inactivity, a modpack list is loaded. This list is then checked against user's installed mods. If a match is found, the app will pull the modpack's (mt) configuration file. This file contains several options, documented in the API section. But the gist of it, is that all mods have a server and client side sha256 checksum, which changes with the file. So when a client side checksum does not match the one on the server, you know it's a different one

Why did you do this?

The main driving point behind this project is to offer support for people with new/ migrated microsoft accounts. The team behind Technic has very clearly stated that they have no intentions on adopting the new OAuth 2.0 system leaving a big chunk of the community behind. Upon looking into this new system, i have decided to support it in my own client.

Secondary driving points


Me and my friends like playing modded Minecraft together, however many of us have new accounts via Microsoft, or a cracked version of the game. This creates a hassle to keep our mod list in sync. With this launcher, we are hoping to eliminate this issue once and for all.
